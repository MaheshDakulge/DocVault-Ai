RECOMMENDED: Option C — Local-First with Cloud Backup. Documents are stored on the device as the primary source of
truth. Cloud is used for backup only. This means the app works 100% offline after the initial scan. Search, browse, view, and
copy fields all work with zero internet. Cloud sync happens silently in the background when online. If the user loses their phone,
documents are recoverable from cloud.
How Local-First Architecture Works
Every document goes through this flow:
SCAN (needs internet once):
User scans image → image bytes sent to Gemini Vision API
Gemini returns: category, fields, date, expiry, confidence
ALL data saved to device immediately:
→ Image file saved to: /data/user/0/com.digisafe/files/docs/{uuid}.jpg
→ Metadata + fields saved to: SQLite database on device
AFTER SCAN (100% offline):
Browse tree view → reads SQLite (offline)
Search 'Aadhaar' → SQLite FTS5 full-text search (offline)
View document → reads local file (offline)
Copy any field → reads from SQLite (offline)
AI assistant basic questions → reads SQLite fields (offline)
BACKGROUND SYNC (when online):
SQLite change log → push to Supabase PostgreSQL
Local files → upload to Supabase Storage
Conflict resolution: device wins (user's edits override cloud)
ONLINE ONLY (needs internet):
First scan (Gemini API call)
AI eligibility engine (Gemini reads docs + queries schemes)
Sharing documents via secure link
Syncing to new device

Offline Search — How SQLite FTS5 Works
Full-Text Search 5 (FTS5) is a built-in SQLite module that creates an inverted index of all text in the database. When a
document is saved, all field values and raw OCR text are inserted into the FTS5 virtual table. Search queries run against this
index in under 5 milliseconds even with 1000+ documents — no internet needed.
SQLite FTS5 table structure:
CREATE VIRTUAL TABLE documents_fts USING fts5(
doc_id, -- UUID reference to documents table
all_text, -- concatenation of ALL field values + raw OCR text
content='documents', content_rowid='rowid'
);
Search query example:
SELECT doc_id, snippet(documents_fts, 1, '', '', '...', 20)
FROM documents_fts
WHERE documents_fts MATCH 'aadhaar OR 1234567890'
ORDER BY rank;
Result: Returns matching doc IDs with highlighted snippet — 100% offline.

6. What Makes DigiSafe Technically Unique
Local-First Architecture
No other Indian document app stores everything on-device first. DigiSafe works 100% offline after the first scan. Search is
millisecond-fast because it never touches the internet.
Single Gemini Call Does Everything
One API call to Gemini Vision replaces three separate systems: OCR engine, document classifier, and field extractor. This
makes the scan pipeline 3x faster and more accurate on regional languages.
SQLite FTS5 Offline Search
Full-text search across all extracted fields works with zero internet. Type '8.44' and find your marksheet. Type 'expire 2025'
and find your passport. No other DigiLocker-alternative has this.
SHA-256 Tamper Detection
Every document gets a cryptographic fingerprint on first scan. If the file is modified after saving, the app detects it
immediately. This is a feature used in forensic systems — not document apps.
